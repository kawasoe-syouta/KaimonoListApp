---
description: 変更をコミットしてリモートにプッシュする
---

現在の変更をコミットし、リモートリポジトリへプッシュしてください。以下の手順に従うこと。

1. 次のコマンドを並行して実行し、現在の状態を把握する:
   - `git status`
   - `git diff`(ステージ済み・未ステージの両方)
   - `git log --oneline -5`(最近のコミットメッセージのスタイルを確認)

2. 変更内容を確認し、関連する変更をステージングする(`git add`)。

3. このリポジトリの慣習に合わせた簡潔で分かりやすいコミットメッセージ(日本語)を作成し、コミットする。コミットメッセージの末尾には必ず以下を含めること:

   ```
   Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>
   ```

4. 現在のブランチをリモートへプッシュする(`git push`)。上流ブランチが未設定の場合は `git push -u origin <branch>` を使う。

5. コミットハッシュとプッシュ結果を報告する。

注意:
- `$ARGUMENTS` が渡された場合は、それをコミットメッセージとして使用する。
- プッシュ前に必ずコミットが成功したことを確認する。
- 機密情報(APIキー、認証情報など)が含まれていないか確認する。
